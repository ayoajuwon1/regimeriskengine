# User Interaction Guide

## Intended User

Institutional investment advisors, CIO support staff, and governance reviewers preparing portfolio risk materials for committee review.

## Operator Workflow

1. Enter the structured portfolio input template.
2. Review the attached market-context evidence and intake-classification summary.
3. Run the analysis workflow.
4. Review the structured outputs for exposures, regimes, vulnerabilities, and governance implications.
5. Complete the validation checklist.
6. Validate, escalate, or return for revision.
7. Release the committee memo only after review completion.

## What The User Should Expect

- the system will not answer general investing questions
- the system will not provide security recommendations
- the system will produce a structured institutional risk package
- the user remains accountable for review and final use

## What The User Must Provide

- portfolio label
- asset allocations and weights
- region / focus by sleeve
- duration
- leverage
- liquidity profile
- investment constraints

## What The User Must Not Do

- use the system as a retail advice engine
- assume output is final without review
- treat missing data as resolved just because the output sounds plausible
